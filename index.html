Cuneiform Digital Library Initiative (CDLI) is a joint project of The University of California Los Angeles (UCLA), The University of Oxford, and The Max Planck Institute for the History of Science, Berlin. It represents the efforts of an international group of Assyriologists, museum curators, and historians of science to make the form and content of cuneiform inscriptions available through the internet dating from the beginning of writing, ca. 3350 BC, until the end of the pre-Christian era.

Web view

This dashboard is supposed to be used by Admins and Editors. It consists of around 20 different admin function controls. The first step was to categorize them followed by deciding each category positioning based on hierarchy in terms of how frequently they are used.

In research conducted via interviews and weekly design meetings, we concluded that different users are handling different sections, the Admin dashboard is accessed through desktop as well as mobile devices, and most of the functions had 2 actions associated with them i.e. adding a new entry and viewing/editing an existing entry thus cards were the best option as a UI element for each function.

Mobile view

Admin dashboard consists of around 20 cards that are bifurcated under six categories but bombarding the user with so many options altogether not just affect the cognition load but also increases scrolling which causes a problem in the navigation to the actual function they wanted to use, therefore I have used accordions for the categories.

Jakob’s law states that users spend most of their time on other sites, which suggest that it is preferred that your site to work the same way as all the other sites they already know. So I have used the circular button at the bottom-right corner because users who are going to the user admin dashboard were very familiar with the mobile interface of apps like Gmail and Slack

## Raw Image Scan

An cuboidal artifact consists of 6 side:

  • Obverse (O)
  • Reverse (R)
  • Left Edge (LE)
  • Right Edge (RE)
  • Top Edge (TE)
  • Bottom Edge (BE)

In order to digitize it, the backend was developed in such a way that an artifact's each side is either scanned using a scanner or photos are taken using a DSLR, each scan can be up to 70 MB. A user with admin dashboard access is supposed to save its name as "ArtifactID_edgeName'' (eg. 247_O where 247 is Artifact ID and O stands for Obverse face.), and after successfully uploading them, a black background is added to each scan and all 6 sides are stacked together.

Problem : There was no reference for the set of rules that should be followed for uploading images and there were substantial chances of error of using wrong naming convention and then that scan has to be renamed and uploaded again..

Zoom

Solution :

Since some sides can be identified even after looking at it, an option to preview and rename even after uploading will save plenty of time and reduce the chances of further mistakes. Also since this task consists of various stages, using a stepper will keep the users updated on the progress and since the whole upload process is dependent on naming conventions, a user must go through a naming convention hence should be the first step before continuing to upload, which is a very redundant process.

Stepper 2: Choose Files
Stepper 3: Edit/start uploading
